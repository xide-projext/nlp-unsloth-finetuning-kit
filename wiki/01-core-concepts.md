## 토대: 행렬의 rank(랭크)

**개념.** 행렬이 실제로 담고 있는 *"독립적인 정보의 개수"*. 행/열이 서로의 조합으로 만들어질 수
있다면, 겉보기 크기보다 실제 정보량은 적습니다.

**비유.** 물감 팔레트에 빨강·파랑 두 원색만 있다고 합시다. 보라·자주는 섞어 만들 수 있지만
초록은 못 만듭니다 — 이 팔레트의 "표현력"은 2차원, 즉 **rank 2**. 색이 다섯 통 있어도 전부
빨강의 농도 차이라면 사실상 **rank 1**.

**예시.** 3×3 행렬인데 둘째·셋째 행이 첫째 행의 2배, 3배라면 독립된 정보는 첫 줄 하나뿐 → rank 1.
칸은 9개지만 정보는 한 줄 분량입니다. (실습 1-(a)에서 직접 확인: 9칸 행렬의 rank가 1로 찍힘.)

> 이 감각이 **LoRA의 전제**입니다. 큰 모델을 특정 과제에 적응시킬 때 생기는 *가중치 변화량(ΔW)이
> 본질적으로 낮은 rank를 가진다*는 관찰이 출발점.

---

## LoRA (Low-Rank Adaptation)

**개념.** 원본 가중치 `W`는 얼려 두고(학습 안 함), 파인튜닝으로 인한 변화량 `ΔW`를 두 개의 얇은
행렬 곱 `B·A`로 근사해 **그 둘만 학습**합니다. 수식: `h = Wx + BAx`.

**수학적 직관.** `d×k`짜리 큰 행렬을 직접 바꾸는 대신, `d×r`과 `r×k`라는 홀쭉한 두 행렬로 쪼갭니다.
`r`(rank)이 작으니 학습 파라미터가 `d·k` → `d·r + r·k`로 급감. 발상은 PCA·SVD처럼 고차원 행렬을
저차원으로 근사하는 것과 같은 계열 — 단, ΔW를 먼저 구해 분해하는 게 아니라 **A·B를 처음부터
직접 학습**한다는 게 핵심입니다.

**비유.** 두꺼운 백과사전(W) 본문은 그대로 두고, 여백에 얇은 포스트잇 메모(B·A)만 붙여 내용을
보정하는 것. 본문을 다시 인쇄하지 않으니 싸고 빠릅니다.

**예시 (실습 1-(c) 실측).** 4096×4096 가중치 = 16,777,216개.
| LoRA rank | 학습 파라미터 | 전체 대비 |
|---:|---:|---:|
| r=8 | 65,536 | **0.39%** |
| r=16 | 131,072 | 0.78% |
| r=32 | 262,144 | 1.56% |
| r=64 | 524,288 | 3.12% |

→ "전체 가중치의 약 1%만 최적화"라는 말의 근거. 실습 1-(d)에서는 `B=0`에서 시작(=원본 보존)해
경사하강으로 목표 ΔW를 따라가는 과정을 직접 학습시켜 봅니다(loss 2.76 → 0.03).

## 양자화 (Quantization)

**개념.** 16-bit 실수로 저장된 가중치를 4-bit 같은 더 낮은 정밀도로 압축. 핵심은 **매핑** —
부동소수점 값의 범위를 더 작은 정수 집합(예: INT4의 16개 구간)에 대응시키는 것.

**비유.** JPEG와 똑같습니다. 1,670만 색을 작은 팔레트로 줄여도 대부분 눈치채지 못하듯, 양자화된
가중치도 출력 품질을 거의 유지합니다. 또는 자의 눈금을 mm→cm로 바꾸는 것: 저장은 가벼워지되
정밀도는 약간 잃습니다.

**한 단계 더 (NF4).** 막 자르지 않습니다. 신경망 가중치가 **정규분포**(0 근처에 몰림)를 따른다는
점을 이용해, 각 4-bit 구간이 분포의 동일한 몫을 담도록 설계한 **NF4(NormalFloat4)** 데이터타입을
씁니다. 값이 몰린 곳에 눈금을 촘촘히 두는 셈.

**예시 (실습 2 실측).**
- 4-bit 균등 눈금: 평균 절대오차 **0.01735**
- 4-bit 분위수 눈금(NF4식): 평균 절대오차 **0.00849** → **51% 오차 감소**
- 7B 모델: 16-bit 14.0GB → 4-bit 3.5GB (**75% 절감**)

## QLoRA = 양자화 + LoRA

**개념.** 얼린 원본을 4-bit(NF4)로 압축해 메모리를 ~75% 줄이고, 그 위에 **LoRA 어댑터만** 정밀하게
(16-bit로) 학습합니다.

**효과.** 예시로 **8GB VRAM에서 3B급 모델 파인튜닝**이 가능. 무료 Colab/로컬 GPU로 과제를 돌릴 수
있게 해 주는 핵심 기술입니다.

**LoRA vs QLoRA — 어느 걸 쓸까?**
| | LoRA | QLoRA |
|---|---|---|
| 원본 저장 | 16-bit | 4-bit(NF4) |
| 메모리 | 더 씀 | ~75%↓ |
| 속도/품질 | 약간 유리 | 거의 동등, 약간 손해 가능 |
| 언제 | VRAM 여유 있을 때 | **VRAM 빠듯할 때(대부분의 학생 환경)** |

> 학생 환경(무료 Colab T4 16GB, 로컬 8–12GB)이라면 **QLoRA가 기본 선택**입니다.

## 📄 더 읽을 거리 — 대표 논문 (이 섹션 개념의 원전)

각 항목은 **논문이 개념을 어떻게 설명/비유하는지 + 직접 든 예시(수치) + 링크**입니다.

### ① LoRA: Low-Rank Adaptation of Large Language Models (Hu et al., 2021)
- **설명 방식**: 친근한 비유 대신, *"큰 모델을 적응시킬 때 생기는 가중치 변화 ΔW가 본질적으로 낮은
  rank를 가진다"*는 **가설**을 세우고 `ΔW = B·A`로 분해해 검증. "과하게 큰 모델은 사실 적은
  자유도로 학습된다"는 직관을 정량화한 글.
- **직접 예시**: GPT-3 **175B에서 학습 파라미터를 10,000배, GPU 메모리를 3배** 줄이면서 full
  fine-tuning과 동등 이상. 일부 층은 **rank r=1~2**로도 충분함을 부분공간 유사도(Grassmann 거리)로
  보임. → 실습 1과 정확히 같은 발상.
- 🔗 <https://arxiv.org/abs/2106.09685>

### ② Intrinsic Dimensionality Explains the Effectiveness of Fine-Tuning (Aghajanyan et al., 2020)
- **왜 보나**: LoRA가 "왜 통하는가"의 직접적 토대.
- **설명 방식**: *"사전학습 모델은 매우 낮은 intrinsic dimension을 갖는다"* — 전체 파라미터 공간 대신
  **무작위 저차원 부분공간**에 투영해 학습해도 거의 같은 성능이 난다는 것으로 '저차원성'을 증명.
- **직접 예시**: RoBERTa를 **단 200개**의 학습 파라미터(무작위 부분공간)만으로 튜닝해 MRPC에서
  full 성능의 **90%** 달성.
- 🔗 <https://arxiv.org/abs/2012.13255>

### ③ QLoRA: Efficient Finetuning of Quantized LLMs (Dettmers et al., 2023)
- **설명 방식**: NF4(4-bit NormalFloat) — *"가중치가 정규분포이니 분포에 맞춰 4-bit 눈금을 배치하면
  정보이론적으로 최적"*이라는 논리(실습 2-(c)의 분위수 눈금이 바로 이 아이디어). +
  double quantization(양자화 상수까지 한 번 더 양자화) + paged optimizer(메모리 스파이크 방지).
- **직접 예시**: **65B 모델을 단일 48GB GPU**에서 튜닝하며 16-bit full 성능 유지. Guanaco가 단일 GPU
  **24시간** 학습으로 ChatGPT의 **99.3%** 도달.
- 🔗 <https://arxiv.org/abs/2305.14314>

### ④ LLM.int8() (Dettmers et al., 2022) — 양자화의 함정
- **설명 방식**: 큰 모델에는 소수의 *'이상치(outlier) 특징'*이 있어 단순 양자화가 무너진다 → 이상치만
  16-bit로 따로 빼고 나머지는 8-bit로 처리(혼합 정밀도 분해).
- **직접 예시**: 약 **6.7B 파라미터** 규모부터 outlier feature가 창발(emergent)해 성능을 좌우.
  "왜 그냥 8-bit로 못 자르나"의 답.
- 🔗 <https://arxiv.org/abs/2208.07339>

### ⑤ GPTQ (Frantar et al., 2022) — 학습 없는 사후 양자화(PTQ)
- **설명 방식**: 이미 학습된 모델을 추가 학습 없이 2차근사 기반으로 3~4bit로 압축. (QLoRA의
  '학습 중 양자화'와 대비되는 '학습 후 양자화'.)
- **직접 예시**: OPT/BLOOM **175B를 약 4 GPU시간**에 3~4bit로 양자화, 단일 GPU에서 175B 추론.
- 🔗 <https://arxiv.org/abs/2210.17323>
