# 기말 프로젝트 과제 안내 — Unsloth 파인튜닝

> 자연어정보분석 · 형식 참고: [edu-llm-colab-unsloth](https://github.com/xide-projext/edu-llm-colab-unsloth)
> 이 과제를 수행하기 위한 학습 자료·실습·리포트 템플릿은 본 저장소(README 참조)에 모두 있습니다.

## 주제

학생은 **자신이 정한 도메인 또는 페르소나**를 선택하여, [Unsloth Studio](http://unsloth.ai/docs/new/studio)를
이용해 **베이스 모델을 직접 파인튜닝한 자신만의 LLM**을 만든다.

이번 프로젝트의 목표는 단순한 AI API 호출이나 프롬프트 엔지니어링이 아니라, **베이스 모델 선택 →
데이터셋 설계 → LoRA/QLoRA 학습 → 베이스 대비 개선 입증 → 배포(export)** 로 이어지는 실제
파인튜닝 파이프라인을 완성하는 것이다.

---

## 프로젝트 주제 예시 (택1, 또는 직접 제안)

아래 중 하나를 선택하거나, 별도 주제를 정했으면 사전에 메일로 연락한다.

| # | 시나리오 | 데이터 출처 예 |
|---|---|---|
| 1 | 특정 **페르소나/말투** 챗봇 (캐릭터·존댓말 레벨·브랜드 톤) | 직접 작성한 대화 수백 개 |
| 2 | **강의자료 Q&A 봇** (한 과목을 통째로 학습) | 강의 PDF → Data Recipes |
| 3 | **사규/매뉴얼 도우미** (사내 규정·제품 매뉴얼 응답) | 사내 문서 DOCX/PDF |
| 4 | **포맷 강제 응답** (항상 JSON/표/특정 템플릿으로 답) | 입력-정답 쌍 |
| 5 | **텍스트 분류·태깅** (감성·주제·의도 라벨링) | 라벨링된 데이터셋 |
| 6 | **요약 전문 모델** (특정 분야 문서 요약 스타일 고정) | 문서-요약 쌍 |
| 7 | **번역/문체 변환** (격식↔비격식, 방언, 쉬운 말 변환) | 병렬 문장 쌍 |
| 8 | **코드 도우미** (특정 라이브러리·사내 컨벤션 특화) | 코드 Q&A |
| 9 | **상담/FAQ 봇** (자주 묻는 질문에 일관된 답) | FAQ 로그 |
| 10 | **도메인 용어 설명가** (전문용어를 쉬운 비유로) | 용어-설명 쌍 |
| 11 | **퀴즈/문제 생성기** (지문에서 문제·정답 생성) | 지문-문항 쌍 |
| 12 | **특정 언어/저자 문체 모방** | 해당 텍스트 코퍼스 |

> 주제 선정·데이터 설계는 [`wiki/03-dataset-design.md`](wiki/03-dataset-design.md) 참조.

## 필수 요구사항

프로젝트는 다음 요소를 **모두** 포함해야 한다.

- **베이스 모델 선택**
  자신의 목표·VRAM 예산에 맞는 베이스 모델을 고르고, *왜 그 모델인지* 근거를 댄다.
  (입문은 8B급 또는 0.5~3B instruct 모델 권장. → [`wiki/04`](wiki/04-unsloth-studio-workflow.md) ①)
- **데이터셋 설계**
  role 태깅 + 베이스 모델의 chat template에 맞춘 데이터를 직접 구성한다. *품질 > 양*.
  (최소 수백 개. PDF/CSV가 있으면 Data Recipes 활용. → [`wiki/03`](wiki/03-dataset-design.md))
- **Unsloth 활용 (실제 학습)**
  Unsloth Studio(no-code) 또는 Unsloth 코드(Colab)로 **실제로 학습을 돌린다.**
  단순 추론·프롬프팅만 한 프로젝트는 인정하지 않는다.
- **LoRA/QLoRA 하이퍼파라미터 설정**
  rank·alpha·learning rate·epoch·target modules를 *근거를 갖고* 설정한다.
  (→ [`wiki/02-hyperparameters.md`](wiki/02-hyperparameters.md))
- **베이스 vs 파인튜닝 비교**
  학습에 쓰지 않은 새 프롬프트로 베이스 모델과 출력을 비교한다(Studio의 Model Arena).
  도메인 질문 + 무관한 질문(망각 체크)을 섞는다.
- **학습 모니터링**
  training loss 곡선을 근거로 과적합/과소적합을 판단하고 대응한 과정을 남긴다.
- **Export & 실행**
  학습 결과를 GGUF 등으로 export하고, 로컬(Ollama 등)에서 실행 가능함을 보인다.

## 제출물

다음 3가지를 제출한다.

### 1. GitHub Repository
오픈소스로 공개된 Repository 링크를 제출한다. README에는 다음을 포함한다.

- 프로젝트 소개 (어떤 도메인/페르소나로 특화했는가)
- 설치 방법 / 실행 방법 (Colab 또는 Studio 절차, 추론 방법)
- 주요 기능 (파인튜닝 모델이 무엇을 더 잘하는가)
- 사용한 기술 스택 (베이스 모델, Unsloth, LoRA/QLoRA, 데이터 출처)
- **파인튜닝 설계 설명** (모델 선택·데이터셋·하이퍼파라미터의 *근거*)
- 베이스 vs 파인튜닝 **비교 결과** (표/스크린샷)
- 실행 화면 또는 스크린샷 (loss 곡선, Model Arena, 추론 예시)

### 2. 프로젝트 보고서
보고서에는 다음을 포함한다. (템플릿: [`report/final-project-template.md`](report/final-project-template.md))

- 선택한 시나리오 / 문제 정의
- 서비스(모델) 대상 사용자
- 베이스 모델 선택과 근거
- 데이터셋 설계 (출처·형식·규모·정제)
- 하이퍼파라미터 설정과 **근거** (논문/문서 인용 → [`wiki/05`](wiki/05-resources.md))
- 학습 과정 (loss 곡선, 과적합/과소적합 판단)
- 구현 결과 (베이스 대비 정량·정성 비교)
- 한계점 및 개선 방향

### 3. 발표 영상
모델 시연 영상을 제출한다. 영상에는 다음을 포함한다.

- 프로젝트 소개
- 데이터셋·학습 과정 요약
- **베이스 vs 파인튜닝 실시간 비교 시연**
- 하이퍼파라미터 선택 근거 설명
- 최종 결과와 개선 방향

## 평가 기준

| 항목 | 비중 |
|---|---|
| 아이디어와 문제 정의 (도메인 적합성) | 20% |
| Unsloth 파인튜닝 활용도 (실제 학습·export) | 20% |
| 데이터셋 설계 & 하이퍼파라미터 근거 | 25% |
| 베이스 대비 개선도 (비교·정량 평가) | 15% |
| GitHub Repository 품질 | 10% |
| 보고서 및 발표 영상 | 10% |

> 점수는 "버튼을 눌렀다"가 아니라 **"왜 이 모델·데이터·하이퍼파라미터를 골랐고, 무엇이 어떻게
> 좋아졌는가"를 근거 있게 설명**하는 데서 갈린다.

## 최소 통과 조건

다음 조건을 **모두** 만족해야 한다.

- 도메인/페르소나 주제를 1개 선택한다. (직접 제안 시 사전 메일 연락)
- 퍼블릭 GitHub Repository를 제출한다.
- **Unsloth로 실제 파인튜닝을 수행**한다. (학습 로그 제시)
- 베이스 모델과 비교해 **변화/개선을 입증**한다.
- 데이터셋·하이퍼파라미터 선택 근거가 보고서에 설명되어야 한다.
- 발표 영상과 보고서를 제출한다.
