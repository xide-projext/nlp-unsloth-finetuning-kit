# 02. 하이퍼파라미터 — 무엇을 만지면 무엇이 변하나

리포트의 "왜 이 값을 골랐는가"를 쓰려면 각 손잡이의 **방향 감각**이 필요합니다.
권장값은 모두 Unsloth 공식 가이드 기준 ([`05-resources.md`](05-resources.md) 참조).

---

## 한눈에 보는 권장값 (출발점)

| 손잡이 | 권장 출발값 | 방향 |
|---|---|---|
| 학습률 (learning rate) | **2e-4** (LoRA/QLoRA) | 너무 크면 발산, 작으면 안 배움 |
| 에폭 (epochs) | **1–3** | 넘기면 과적합 |
| LoRA rank (r) | **8–32** (범위 4–64) | 클수록 표현력↑·과적합 위험↑ |
| LoRA alpha (α) | **α ≥ r**, 흔히 **α = 2r** | 어댑터 반영 강도 |
| target modules | **All** (attn q/k/v/o + MLP gate/up/down) | 많이 걸수록 성능↑ |
| effective batch | **16 내외** (batch × grad-accum) | 클수록 안정·메모리↑ |
| LoRA dropout | 0–0.1 | 과적합 방지용 |

> 강화학습(DPO·GRPO)은 학습률이 훨씬 작습니다(예 **5e-6**). 일반 SFT와 혼동 금지.

---

## 학습률 (Learning Rate)

**개념.** 매 업데이트에서 가중치를 얼마나 크게 움직일지 정하는 *보폭*. `W ← W − η·∇L` (η가 학습률).

**비유.** 안개 낀 산에서 가장 낮은 골짜기를 찾아 내려가는 상황. 보폭(η)이 너무 크면 골짜기를
건너뛰며 좌우로 진동하고, 너무 작으면 한나절이 지나도 못 내려갑니다.

**예시.** 일반 LoRA/QLoRA는 **2e-4(0.0002)** 를 출발점으로, 강화학습은 5e-6 정도.
warmup(처음 몇 스텝 동안 lr를 0→목표로 서서히 올림) + cosine decay를 흔히 같이 씁니다.

---

## 에폭 (Epoch)

**개념.** 전체 학습 데이터를 처음부터 끝까지 몇 번 통독하는가.

**비유.** 시험 대비 교재 회독. 1회독은 설익고, 10회독이면 문제와 답을 통째로 외워(**과적합**)
변형 문제에 약해집니다.

**예시.** **1–3 에폭** 권장. instruction 데이터셋에서 3회를 넘기면 효용은 줄고 과적합 위험만 커집니다.

---

## LoRA rank (r)

**개념.** 어댑터 `B·A`의 두께 = 표현력. ([`01-core-concepts`](01-core-concepts.md)의 rank 참조.)

**방향.** 작은 모델·복잡한 데이터셋일수록 r을 키웁니다. 너무 크면 학습 파라미터·과적합·메모리가
같이 늘어납니다. **8 또는 16에서 시작**해 결과를 보고 조정하세요.

---

## Alpha (α, 스케일링 계수)

**개념.** 학습한 어댑터를 원본에 얼마나 세게 반영할지 정하는 *볼륨 노브*. 실제 적용은 `(α/r)·BA`.

**비유.** 커피의 시럽 펌프 수. 같은 시럽(어댑터)이라도 몇 펌프 넣느냐로 맛이 달라집니다.

**예시.** 경험칙으로 **α = 2r** (r=16 → α=32). 최소한 **α ≥ r**. (실습 1-(d)에서 `scale=α/r=2`로 사용.)

---

## Batch size & Gradient Accumulation

**개념.** 가중치를 한 번 업데이트하기 전에 몇 개의 예시를 보고 평균 방향을 정할지.

**비유.** 여론조사 표본 크기. 1명만 물으면 그 사람 취향(노이즈)에 휘둘리고, 여러 명 평균을 내면
방향이 안정됩니다. **그래디언트 누적**은 GPU 메모리가 작아 큰 표본을 한 번에 못 볼 때, 작은 배치를
여러 번 모아 큰 배치 효과를 내는 기법.

**예시.** `batch 2 × accum 8 = 유효 배치 16`. VRAM이 부족하면 batch를 줄이고 accum을 늘려 유효
배치를 유지하세요.

---

## Target Modules (적용 대상 모듈)

**개념.** 트랜스포머의 어느 부품에 LoRA를 붙일지. attention의 q·k·v·o 프로젝션 + MLP의
gate·up·down 프로젝션이 후보.

**비유.** 자동차 튜닝에서 엔진만 손볼지, 엔진+서스펜션까지 다 손볼지의 선택.

**예시.** attention만 거는 것보다 **MLP 포함 전체(All)** 가 실험적으로 더 우수.
Unsloth 프리셋은 보통 All을 기본으로 잡아 줍니다.

---

## 과적합 vs 과소적합 — training loss 읽는 법

**개념.** 과적합 = 학습 데이터에만 과하게 맞아 새 데이터에서 무너짐. 과소적합 = 충분히 못 배워
기본도 못 풂.

**비유.** 기출만 통째로 외운 학생(과적합) vs 공부량 자체가 부족한 학생(과소적합). 둘 다 실전 점수는
낮습니다.

**진단 (Studio의 실시간 loss·gradient norm·GPU 그래프 활용):**

| 증상 | 해석 | 처방 |
|---|---|---|
| train loss가 안 떨어짐 | 과소적합 | lr↑, epoch↑, r↑ |
| train loss↓인데 출력 품질 나쁨 | 데이터 문제 | 데이터 품질·형식 점검 |
| train loss 거의 0인데 새 입력에 이상 | 과적합 | epoch↓, dropout↑, 데이터↑ |
| loss가 들쭉날쭉 폭발 | lr 과대 | lr↓, warmup 추가 |

> **목표**: 하이퍼파라미터를 조정해 과적합·과소적합을 막으면서 정확도를 올리는 것.
> 리포트에는 loss 곡선 스크린샷 + "이 그래프를 보고 무엇을 어떻게 바꿨다"를 함께 적으세요.

## 📄 더 읽을 거리 — 대표 논문 (손잡이 값의 근거)

### ① LoRA (Hu et al., 2021) — rank·대상모듈 ablation
- **설명 방식**: "어느 weight 행렬에, rank 얼마로 거는 게 좋은가"를 ablation 표로 직접 비교.
- **직접 예시**: 같은 파라미터 예산이면 q,v 한두 행렬에 큰 rank를 거는 것보다 **여러 행렬에 작은
  rank로 분산**하는 게 유리. rank를 키워도 성능이 **금방 포화**(작은 r로 충분) → 본 위키의
  "r=8~16에서 시작, target=All" 권장의 근거.
- 🔗 <https://arxiv.org/abs/2106.09685>

### ② rsLoRA: A Rank-Stabilization Scaling Factor for LoRA (Kalajdzievski, 2023) — alpha의 진실
- **설명 방식**: 표준 스케일 `α/r`은 **rank가 커질수록 gradient가 붕괴**한다(클수록 업데이트가
  억눌림)는 걸 수학적으로 보이고, 출력·gradient의 norm이 rank에 무관하게 일정하도록 **`α/√r`** 를
  유도.
- **직접 예시**: `α/√r`을 쓰면 rank를 키울수록 성능이 계속 좋아짐(compute↔성능 트레이드오프 확보).
  표준 `α/r`에서는 정체. → Studio/PEFT의 **`use_rslora`** 옵션. 본 위키의 `α=2r` 경험칙은
  **작은 r에서만 안전**하고, 큰 r을 쓸 거면 rsLoRA를 고려하라는 뜻.
- 🔗 <https://arxiv.org/abs/2312.03732>

### ③ DoRA: Weight-Decomposed Low-Rank Adaptation (Liu et al., 2024) — LoRA의 개량
- **설명 방식**: 가중치를 **크기(magnitude)와 방향(direction)** 으로 분해하고, 방향만 LoRA로 학습 →
  full fine-tuning에 더 가까운 학습 거동을 얻는다는 분석.
- **직접 예시**: 같은 파라미터 예산에서 여러 벤치마크에서 LoRA를 상회. 추론 비용 추가 없음.
  (Studio에 옵션이 있으면 비교 실험 거리로 좋음.)
- 🔗 <https://arxiv.org/abs/2402.09353>
